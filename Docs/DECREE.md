# 칙령 (Decree)

이 문서는 칙령 시스템의 현재 상태와 재도입 기준을 정리합니다.

## 문서 범위/분리 기준

- 전체 턴/상태머신: `Docs/GAME_STRUCTURE.md`
- 이 문서: 칙령 구현 상태, 추후 연결 시 요구 사항

## 현재 상태 (v0)

- 현재 코드/데이터 로더에는 칙령 전용 테이블 로딩이 연결되어 있지 않습니다.
- 런타임 상태에도 칙령 인벤토리/사용 파이프라인이 구현되어 있지 않습니다.
- 즉, v0 기준으로 칙령은 개념만 유지되고 실제 플레이에서는 비활성입니다.

## 용어 유지 목적

- 문서/기획 용어 사전에서는 `칙령(Decree)`를 유지합니다.
- 파워 예산 문서에서 `P_item` 축을 별도로 관리하기 위한 자리로 남겨둡니다.

## 재도입 시 최소 요건

1. 정적 데이터 테이블 추가 (`Decrees.json` 또는 동등 파일)
2. 런타임 보유/획득/소모 상태 모델 추가
3. 사용 입력/UI(슬롯 또는 인벤토리) 추가
4. 효과 적용 경로를 `EffectBundle` 체계와 정합되게 연결
5. 스킬/요원 선택 단계와 입력 충돌 우선순위 확정
