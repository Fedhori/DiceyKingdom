# 게임 아이디어 백로그

이 문서는 떠오르는 아이디어를 빠르게 기록하고, 구현 전 후보를 정리하기 위한 백로그입니다.
구조화된 현재 설계 기준은 `Docs/GAME_STRUCTURE.md`를 우선합니다.

## 플레이테스트 인사이트 로그

### 2026-02-09 밤 플레이테스트 요약

- 관찰 1: "주사위 자체만 분배" 루프는 기대보다 흥미가 낮았습니다.
  - 주사위 수가 적을 때:
    - 시너지 밀도가 낮고 결과가 단순 합산으로 수렴
    - 실패 체감이 "도전"보다 "불쾌한 악운"으로 인식됨
  - 주사위 수가 많을 때:
    - 정보량/선택지가 과도해져 선택 피로 유발
    - 개별 재미보다 기대값 계산 작업으로 플레이가 경직됨
- 관찰 2: "결과가 불확실한 자원"을 분석/직관/계산으로 분배하는 핵심 컨셉은 여전히 유효 가능성이 있습니다.
- 가설 1: 분배 단위를 주사위 단일체가 아닌 "모험가" 패키지로 바꾸면 의사결정 밀도를 높이면서도 정보량을 통제할 수 있습니다.
  - 예시 구성: 굴림 주사위 수, 고유 효과, 장비 슬롯
- 가설 2: 끊김 없는 단일 누적 스테이지는 역전 서사를 만들기 어렵고, 파워 열세 구간에서 붕괴가 급격합니다.
  - 개선 방향: 스테이지 단위 진행 + 적(기존 Situation) 액션의 준비시간/발동 구조
- 프로세스 결론:
  - "세세한 기획 선작성"보다 "빠른 플레이어블 v0 빌드"로 핵심 가설을 먼저 검증해야 합니다.
  - 세부 수치/규칙 확정은 v0 검증 이후로 지연합니다.

### v0 검증 목표(우선순위)

- 목표 문장:
  - "결과가 불확실한 자원을 플레이어가 분석/직관/계산으로 분배하는 행위가 반복적으로 재미를 제공하는가?"
- 1차 합격 기준(확정):
  - 플레이 중/종료 후 "재미 신호"가 명확히 감지되는가
  - 약 `10`회 플레이에서도 재미 신호가 거의 없으면 아이디어 중단 후보로 판정
- 1차 검증 포인트:
  - 턴당 선택 피로(과부하) 없이 의미 있는 고민이 발생하는가
  - 악운 체감이 불쾌감으로 치우치지 않고 대응 가능성으로 인식되는가
  - 빌드/단위 조합이 단순 기대값 계산을 넘어선 질적 차이를 만드는가

### v0 가드레일(초안)

- 분배 단위가 모험가일 경우, 단위 수는 `4`로 고정합니다.
- 이번 v0 빌드는 모험가 <-> 스킬 시너지 검증이 목적이므로, 스킬 최소 세트 3종을 기본 보유로 시작합니다.
  - 원하는 모험가 리롤(쿨다운 `1`)
  - 원하는 주사위 눈 +1(쿨다운 `1`)
  - 지정 몬스터에 피해 2(쿨다운 `2`)
- 스킬 규칙은 "개별 쿨다운 + 각 스킬 턴당 1회 사용"으로 고정합니다.

### 실험 전략 결정(확정)

- 이번 실험은 `선택지 A(분배 단위만 전환)`로 진행합니다.
  - 분배 단위: 모험가
  - 진행 구조: 연속 턴 보드 유지 + 보드 전멸 시 다음 스테이지 프리셋 랜덤 소환
- 실험 빌드 분리는 이번 사이클에서 수행하지 않습니다.
  - 결정 이유: 우선 분배 단위 변경 효과를 빠르게 확인하고, 스테이지 구조는 다음 사이클에서 별도 전환 가능
- 수용하는 리스크:
  - 연속 누적 구조 고유 문제(말리는 런의 급격한 붕괴)가 남아 결과를 일부 왜곡할 수 있음
- 완화책:
  - 플레이테스트마다 "재미 신호 약화 원인 태그"를 수동 기록(코드 로그 기능은 v0 범위에서 제외)
  - 권장 태그: `과부하`, `악운`, `무기력`, `규칙이해`, `템포저하`

### v0 런 파라미터 잠금(확정)

- 초기 `Stability`: `20`
- 초기 `Gold`: `0`
- 몬스터 스테이지 프리셋 풀: `3`개(사전 제작)
- 각 프리셋 위협 예산: 동일 값으로 고정
- 시작 시 프리셋 3개 중 랜덤 소환
- 턴당 신규 몬스터 생성 수: `0`
- 보드 전멸 시: 다음 턴 `P1`에서 프리셋 3개 중 랜덤 재소환
- 보드 최대 몬스터 수: 상한 없음

## 설계 결정 로그

### 2026-02-09 - 롤 후 할당 방식 기각

- 대상 변경안:
  - 기존: 모험가를 적에 할당 -> 주사위 롤 -> 조정 -> 최종 피해 적용
  - 제안: 주사위 롤 -> 조정 -> 최종 피해를 적에 할당
- 기각 사유:
  - 롤 후 할당은 인지부하를 낮추는 장점이 있으나, 할당 기반 상호작용의 재미를 약화시킴
  - 예: 특정 태그 대상 보너스, 협공 시 보너스 등 "사전 할당"이 만들어내는 긴장/보상이 약해짐
  - 롤 후에도 할당 전/후 수치가 크게 달라질 수 있어, 오히려 "어디에 넣으면 얼마나 오르는지"를 반복 확인하게 되어 인지부하가 다시 증가함
  - 결론적으로 v0 방향에서는 "사전 할당 + 롤 후 조정" 구조를 유지

### 2026-02-09 - 배치 경고/Dead 처리 단순화

- 확정 규칙:
  - 체력 0 이하 `Enemy`는 즉시 보드에서 제거
  - 미배치 경고는 단순 확인 모달로 처리(강제 차단 없음)

### 2026-02-09 - P2 입력/시각화 규칙 확정

- 확정 규칙:

  - 빈 공간 드롭 시 해당 모험가 배치 해제
  - v0는 모바일 UX를 고려하지 않음(데스크톱 우선)

  - 배치 관계를 직관적으로 읽게 하여 상황 파악 속도를 높임

### 2026-02-09 - P3/P4 실행 규칙 확정

- 확정 규칙:

- 의도:

## 시스템 아이디어

- `[채택]` 안정도(왕국 체력) 시스템
  - 0 이하가 되면 멸망(게임 오버)
  - 상황 실패/디버프로 감소, 보상/행동으로 회복
- `[채택]` 주사위 분배 기반 해결 구조
  - 매 턴 보유 주사위를 상황 카드 슬롯에 할당 후 굴림
  - 상황마다 주사위 최대 할당 개수 제한 가능
- `[채택]` 주사위 강화 시스템
  - 주사위당 강화 1개 장착
  - 예시: 모든 눈 +1, 리롤 1회 보장, 2회 굴려 높은 눈 사용
- `[채택]` 스킬(Skill) 시스템
  - 액티브 효과 + 턴 쿨다운
  - 전술적 보정(리롤/눈 보정/남은 턴수 조절 등)
- `[채택]` 구조물(Structure) 시스템
  - 런 동안 유지되는 영구 패시브
  - 빌드 방향과 장기 생존력 강화
- `[아이디어]` 오버킬 매커니즘
  - 슬롯을 초과해서 달성한 수치(오버킬)에 반응하는 트리거/보상 시스템
  - 방향성: 고점 주사위 운용과 시너지 빌드의 보상 체감 강화
  - 세부 규칙은 미정(즉시 보상형 vs 누적형 vs 조건형)

- `[아이디어]` 묘지기(모험가)
  - 기믹: 배정된 적이 이미 처치된 경우, 다음 적으로 자동 재지정 후 공격
  - 의도: 과잉 화력으로 인한 공중분해 손실을 부분 완화
  - 리스크: 배치 순서 전략의 의미를 약화하거나 난이도 하향으로 이어질 수 있음
  - 상태: `[아이디어]`

  - 리스크: 기본 규칙의 "상황 잠금" 긴장감을 약화할 수 있음
  - 개선안:
    - 쿨다운/사용 조건을 부여해 상시 재배치화 방지
    - 같은 턴 내 1회 또는 상황당 1회 제한
  - 상태: `[아이디어]`

- `[보류]` 무쿨다운 비용형 Skill
  - 기믹: 쿨다운 대신 골드를 소모해 즉시 효과 발동
  - 의도: 비상 상황에서 골드를 전술 파워로 교환하는 선택 제공
  - 리스크: 골드 수급이 안정되면 사실상 상시 버프가 될 수 있음
  - 개선안:
    - 글로벌 하드 제한 대신, 개별 스킬별 비용 구조로 위험도 제어
    - 예시: `용병단장`은 사용 시마다 비용이 1씩 증가
    - 특정 태그 상황에서만 사용 가능하게 설계해 범용성 과다를 방지
  - 보류 사유: v0 정책(모든 스킬은 개별 쿨다운 + 턴당 1회)과 충돌
  - 상태: `[보류]`

- `[아이디어]` 용병단장 (Skill)
  - 기믹: 선택한 주사위 눈 +1, 사용 시 골드 소모
  - 의도: 골드를 즉시 전투력으로 치환하는 전술형 버튼
  - 리스크: 골드 수급과 결합 시 반복 사용 효율이 과도해질 수 있음
  - 개선안:
    - 사용할수록 비용 증가(누진)
    - 특정 턴/상황에서만 추가 보정 가능
  - 상태: `[아이디어]`

- `[채택]` 스킬 용어 통일(영문 우선)
  - 기믹: 스킬을 "인물" 테마로 묶는 명명 규칙 적용
  - 확정 용어: `Skill`
  - 의도: 세계관 몰입과 인지 일관성 강화
  - 리스크: 용어가 추상적이면 기능 직관성이 떨어질 수 있음
  - 개선안:
    - 카테고리명은 `Skill`로 고정하고
    - 카드 타이틀/아이콘에서 기능 타입을 보조 표기
  - 상태: `[채택]`

- `[아이디어]` 스킬 슬롯 가중치 시스템
  - 기믹: 스킬마다 점유 슬롯이 다르고, 플레이어는 총 슬롯 한도 내에서 구성
  - 의도: 단순 개수 제한보다 빌드 의사결정 밀도 강화
  - 리스크: UI/인지 복잡도 증가, 초반 진입장벽 상승
  - 개선안:
    - 초반에는 슬롯 비용 1 위주로 시작해 점진적으로 고비용 스킬 해금
    - 덱빌딩 UI처럼 현재 점유/잔여 슬롯을 항상 시각화
  - 상태: `[아이디어]`

- `[아이디어]` 국경수비대 (Structure)
  - 효과: 모든 `침략` 태그 상황의 시작 목표치 -1

- `[아이디어]` 최후의 의지 (Structure)
  - 기믹: 안정도가 0 아래로 내려가도 1턴 유예 상태로 생존
  - 의도: 역전 기회 제공, 극단 턴에서의 드라마 강화, 더 과감한 리스크 선택 유도
  - 리스크: 사실상 추가 목숨처럼 작동해 난이도 체감을 왜곡할 수 있음
  - 개선안:
    - 획득 난이도를 높게 설정해 접근성을 제한
    - 런당 1회 한정
    - 유예 턴 종료 시 안정도 1 이상 회복 실패하면 즉시 게임 오버
  - 상태: `[아이디어]`

- `[아이디어]` 은행 (Structure)
  - 기믹: 골드를 `-N`까지 대출(적자) 가능하게 확장
  - 의도: 단기 부채를 통한 고위험 고보상 운영 허용
  - 리스크: 대출이 너무 강하면 사실상 공짜 골드로 변질
  - 개선안:
    - 효과 텍스트는 단일 규칙(대출 한도)로 유지해 직관성을 우선
    - UI에 현재 골드와 대출 한도(`-N`)를 상시 표시
    - 중복 획득 시 대출 한도는 중복 적용(누적)
  - 상태: `[검토중]`

- `[채택]` 구조물 용어 전환(영문 우선)
  - 기믹: "유산"을 구조물 명칭으로 교체
  - 확정 용어: `Structure`
  - 의도: 왕국 경영 판타지와 시스템 의미를 더 직관적으로 연결
  - 리스크: 용어가 너무 일반적이면 스킬/칙령과 경계가 흐려질 수 있음
  - 개선안:
    - 카테고리 용어 + 서브타입(예: Military, Civic, Faith) 동시 표기
  - 상태: `[채택]`

- `[아이디어]` 용병단 고용서 (Decree)
  - 대상: `침략` 태그 상황만 사용 가능
  - 효과: 해당 상황 목표치 -4
  - 테마: 칙령/문서형 소모품

- `[아이디어]` 웅장한 동상 (패시브)
  - 효과: 매 3턴마다 안정도 +1
  - 획득처: 거대 프로젝트 완료 보상

## 진행 예시 메모

- 플레이어 기본 상태 예시:
  - 주사위 4개(기본 3, 강화 1)
  - 스킬 3개(리롤 CD1, 눈 +1 CD1, 몬스터 피해 2 CD2)
  - 구조물 1개(이번 턴 최저 눈 +1)
- 전술 포인트:
  - 단기적으로 실패를 허용하고 핵심 실패를 지연시키는 선택
  - 고가치 상황 우선 해결 vs 안정도 보호용 행동 선택

## 오픈 질문

- 스테이지 단위 + 준비시간 액션 구조를 2차 실험에 언제 전환할지
- 2차 사이클 난이도 상승 규칙(신규 생성, 몬스터 강화, 액션 압박 중 우선 축)
- v0 프리셋 3개의 실제 조합안 확정
- 추가 몬스터 2종(`bandit_smugglers`, `siege_shaman`)의 수치 튜닝 기준 확정
- 타입 태그별 전용 디버프/보상 풀 구성
- 성장 보상 주기(몇 턴마다 강화/스킬/구조물 선택 기회 제공할지)
- 골드의 사용처(상점, 즉시 회복, 확률 이벤트 등)
- 오버킬 보상 규칙(남는 수치 이월, 추가 골드 획득, 전용 트리거 여부)
- 증가형 몬스터 위협의 누적 강도 곡선과 완화 장치(상한/감소/정지) 기준
- 은행형 대출 한도(`N`) 값 확정
